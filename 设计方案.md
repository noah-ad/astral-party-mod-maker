# 吉星派对 立绘 Mod 制作器 — 设计方案

> 面向普通玩家的、傻瓜式的吉星派对（Astral Party）立绘 Mod 制作 / 分享工具。
> 核心交付物是「**图包**」：一个只包含「改过的立绘 + 定位信息」的小文件。

---

## 1. 产品定位

不是简单的"替换器"，而是一套**图包生态**，三大能力：

| 能力 | 说明 |
|------|------|
| **浏览 / 替换** | 显示游戏当前立绘缩略图墙，玩家选中某张 → 拖入自己的图替换 |
| **导出图包** | 把「只改过的」立绘打包成一个图包文件，方便分享（体积小，不含没改的） |
| **导入图包** | 读图包 → 自动识别对应 bundle → 写回替换（一键套用别人的 mod） |

图包里记录了每张图对应的 bundle / 纹理标识，导入时**精确定位**，不依赖"角色名识别"。
角色名识别只用于"浏览时方便人找图"，属于后续增强。

## 2. 技术事实（已勘探确认）

- 游戏引擎：**Unity 2021.3.45f2** + Addressables
- 资源：游戏目录 `StreamingAssets/aa/StandaloneWindows64` 下 **5110 个 `.bundle`**，文件名是 hash
- bundle 内是标准 **Texture2D**，贴图数据走 `.resS`（StreamData 外置）
- 两个安装位置：
  1. 游戏目录 `StreamingAssets/aa/StandaloneWindows64`（主资源）
  2. 热更 DLC：`%AppData%\..\LocalLow\feimo\AstralParty_CN\com.unity.addressables\AssetBundles`（部分皮肤）
- 已验证：**改好整包 bundle 直接覆盖即可生效**（你之前手动做的 mod 证明没有强 CRC 拦截）

## 3. 技术栈

- **C# / .NET 8 / WinForms**（Windows-only，原生拖拽 + 图片墙，最终打包成自带运行时的单 exe，玩家无需装任何环境）
- 直接复用本地库（来自 uabea-windows，免走 NuGet）：
  - `AssetsTools.NET` — 读写 bundle / serialized file
  - `AssetsTools.NET.Texture` + `AssetRipper.TextureDecoder` — Texture2D ⇄ 像素
  - `SixLabors.ImageSharp` — PNG 编解码 / 缩放缩略图
  - `classdata.tpk` — 类型库（bundle 无 typetree 时兜底）

## 4. 图包格式 `.jxpack`（本质是 zip）

```
我的立绘包.jxpack  (zip)
├── manifest.json
└── images/
    ├── 0.png
    ├── 1.png
    └── ...
```

`manifest.json`：
```json
{
  "format": 1,
  "name": "我的妹卡立绘包",
  "author": "作者名",
  "game": "AstralParty",
  "createdAt": "2026-06-27T12:00:00Z",
  "entries": [
    {
      "bundle": "025c3d36156e6922dc20b3a3fb8d5025.bundle",
      "location": "streaming",        // streaming | locallow
      "pathId": 123456,                // Texture2D 在 serialized file 里的 PathID
      "textureName": "xxx_tex",
      "origWidth": 1024,
      "origHeight": 1024,
      "image": "images/0.png",
      "label": "（可选：角色名/备注）"
    }
  ]
}
```

- **只存改过的图** → 图包小
- 导入时按 `bundle + pathId` 精确定位，把 `image` 写回该 Texture2D，重打包 bundle

## 5. 软件界面（MVP）

```
┌─────────────────────────────────────────────────────────┐
│ [游戏路径: D:\steam\...\StandaloneWindows64   ] [⚙设置]   │
│ [搜索🔍] [仅看立绘▾] [导入图包] [导出图包] [一键还原]      │
├──────────────────────────┬──────────────────────────────┤
│  缩略图墙(异步加载/分页)   │   选中图详情                   │
│  ┌──┐┌──┐┌──┐┌──┐         │   原图 ↔ 替换后 对比预览        │
│  │  ││  ││★ ││  │         │   尺寸/格式/bundle             │
│  └──┘└──┘└──┘└──┘         │   [拖入新图替换] [还原此张]     │
│  ★=已改                    │                               │
└──────────────────────────┴──────────────────────────────┘
```

操作流：选图 → 拖入新图 → 标记为"已改" → 应用到游戏（自动备份原 bundle）/ 导出图包。

## 7. 风险与对策

| 风险 | 对策 |
|------|------|
| 写回贴图格式/压缩不对，游戏崩 | 阶段 0 反向验证；保持原 TextureFormat 重编码 |
| 5110 bundle 扫描慢 | 增量索引 + 缩略图缓存到本地 |
| catalog CRC 校验（个别 bundle） | 已验证主资源直接覆盖可行；若遇校验用 addrtool 补 catalog |
| 误操作损坏游戏 | 所有写入前自动备份，提供一键还原 |
